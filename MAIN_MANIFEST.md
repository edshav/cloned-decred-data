# Спецификация Monetarium Explorer

Настоящий документ определяет требования к функциональности и интерфейсу обозревателя блоков Monetarium, построенного на базе форка `dcrdata`.

---

## 1. Технические основы и ограничения

### Поддержка токенов

- Система должна обеспечивать корректную работу и отображение до **255 типов монет SKA**.

### Разрядность и точность данных

- **Монеты VAR:**
  Используют точность 8 знаков в целой части и 8 знаков после запятой (аналогично DCR).

- **Монеты SKA:**
  Имеют повышенную разрядность — **15 знаков в целой части и 18 знаков после запятой**.

- **Особенность реализации:**
  Стандартные типы данных Go не поддерживают такую точность, поэтому для всех расчетов на бэкенде используется **специализированный модуль большой разрядности**.

---

## 2. Стандарты интерфейса (UI)

- **Правило трех цифр (Главная страница):**
  Для всех типов монет (VAR и SKA) на главной странице суммы отображаются с использованием **только трех значащих цифр** и буквенных сокращений (K, M, B, T).

- **Страницы деталей:**
  В деталях блоков и транзакций отображаются **все знаки после запятой** (8 для VAR и 18 для SKA).

- **Адаптивность:**
  Интерфейс строится по принципу **mobile-first** с обязательной поддержкой **темной темы**.

## 3. Главная страница

### 3.1. Таблица Latest Blocks

Таблица предназначена для отображения списка записей с агрегированными значениями и связанными транзакциями с возможностью раскрытия деталей на уровне строки.

### 3.2. Voting

### 3.2.1. Vote VAR Reward
- 0.00000000 VAR/VAR per last block
- 0.00% per 30 days
- 0.00% per year

### 3.2.2. Vote SKA Reward
- Для каждого типа монеты (SKA-n):
  - 00.000000000000000000 SKA-n/VAR per last block
  - 00.000000000000000000 SKA-n/VAR per 30 days
  - 00.000000000000000000 SKA-n/VAR per year

### 3.3. Mining

### 3.3.1. PoW VAR Reward
- Заголовок индикатора: "Next Block VAR Reward Reduction"
- 0.00000000 VAR per last block

### 3.3.2. PoW SKA Reward
- Для каждого типа монеты (SKA-n):
  - 00.000000000000000000 SKA-n per last block

### 3.4. Supply

### 3.4.1. VAR Coin Supply
- Объем эмиссии монет VAR
- 15,093,763 VAR (of ~47M)

### 3.4.2. SKA Coin Supply
- Для каждого типа монеты (SKA-n):
  - In Circulation: 900000000000000 SKA-n
  - Total Issued: 900000000000000 SKA-n
  - Total Burned: 900000000000000 SKA-n

### 3.5. Mempool

### 3.5.1. Таблица Mempool
- В колонке **Type** текущие названия транзакций сохраняются, однако транзакции типа Regular переименовываются в **VAR**, а транзакции простого перечисления монет SKA обозначаются как **SKA-1, SKA-2** и т.д.
- Название колонки DCR заменяется на **Coins** или **Amount**, отображающее количество монет в транзакции.

### 3.5.2. Визуализация распределения места в блоке
- Вместо стандартных элементов реализована полоса-индикатор, представляющая максимально допустимый размер блока (393,216 байт).
- Полоса делится на сегменты, соответствующие пропорциональному размеру транзакций типов VAR, SKA-1, SKA-2 и т.д.

#### Цветовая индикация сегментов:
- **Серый**: Незанятое место в блоке.
- **Зелёный**: Транзакции данной монеты занимают объем, не превышающий гарантированную долю.
- **Жёлтый**: Суммарный размер транзакций превышает гарантированную долю, но все транзакции будут включены за счет неиспользованного пространства других типов монет.
- **Красный**: Транзакции превышают гарантированную долю и дополнительное пространство отсутствует (не все транзакции будут включены).

#### Правила распределения места:
- **VAR**: Гарантированно выделяется 10% общего объема блока.
- **SKA**: Остальные 90% блока делятся поровну между всеми существующими типами монет SKA.
  - 1 тип SKA: 90% гарантии.
  - 2 типа SKA: по 45% гарантии.
  - n типов SKA: по 90/n % гарантии.
